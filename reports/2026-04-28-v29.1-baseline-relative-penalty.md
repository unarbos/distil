# 2026-04-28 v29.1 — Per-axis baseline-relative penalty + KL probe skill diversification

**Status:** SHIPPED 2026-04-28 23:30 UTC. Composite ranking now reflects "stay
above Qwen-4B-base" instead of "absolute pass-frac on procedural items"; the KL
probe samples 20 % from held-out-skill prompts instead of 10 %.

## TL;DR

Every king from 2026-04-17 → 2026-04-28 climbed validator composite while
**regressing** below Qwen3.5-4B base on the held-out canary. v29.1 closes that
loop with two structural changes:

1. **`composite.py` — per-axis baseline-relative penalty.** For each enabled
   bench axis (math/code/reasoning/ifeval/mbpp/aime/tool_use/long_context/
   robustness), the score is docked by `α × (ref - student)` whenever the
   student regresses below the same-round Qwen-4B-base reference. With
   `α = 1.5` a 10 pp regression docks the axis by 15 pp, so a king that
   drops -7 pp on math vs base loses -10.5 pp on `math_bench` inside `worst()`.
   Models that tie or beat base on an axis are unchanged. Reference itself is
   never penalized against itself (it's the anchor by definition).
2. **`eval/private_pool.py` + `state/private_prompt_pool.json` — KL probe skill
   diversification.** Default private-pool fraction raised 10 % → 20 %; pool
   extended from 198 → 278 prompts spanning gsm8k-style word problems,
   humaneval-style algorithm tasks, BBH-style multi-step reasoning, and
   IFEval-style structure constraints. A model that has dropped these skills
   diverges from the teacher more on these prompts than on plain ClimbMix
   continuations, so KL itself now penalizes capability regression.

## Pre-existing defenses that didn't catch it

- **`_baseline_floor_dethrone_veto`** (10 pp absolute floor on transfer axes):
  blocks DETHRONE if a challenger regresses below base. Doesn't dock the king
  itself; if the king is already below base and no challenger is meaningfully
  better, the gate freezes the crown without selecting a better model.
- **`king_canary_streak`** (held-out gap > 5 pp for 2 rounds): waives the
  composite-floor veto for a regressing king. Necessary but not sufficient —
  it requires `state/benchmarks/uid_<king>.json` to exist and be recent, which
  has gaps (the 04-27 kings ran with token-truncation bugs producing 0/0 on
  some axes, suppressing the canary signal).
- **`king_health` telemetry** (worse_than_base, below_floor): dashboard
  signal only, doesn't gate.

The common gap: each of these acts at promotion time, not at scoring time. The
composite RANKING itself was unchanged; a king that's below base on every axis
still scored highly on the validator's per-axis pass-frac because the items are
generally easier than evalscope's items.

## What v29.1 changes

### 1. Per-axis baseline-relative penalty (composite.py)

```python
BASELINE_RELATIVE_PENALTY_ENABLED = 1
BASELINE_RELATIVE_PENALTY_ALPHA = 1.5
BASELINE_RELATIVE_PENALTY_AXES = frozenset({
    "math_bench", "code_bench", "reasoning_bench", "ifeval_bench",
    "aime_bench", "mbpp_bench", "tool_use_bench",
    "long_context_bench", "robustness_bench",
})
```

For each axis in the allow-list:

```python
def _apply_baseline_relative_penalty(axis_value, reference_value):
    if axis_value >= reference_value:
        return axis_value           # parity / above ⇒ full credit
    gap = reference_value - axis_value
    return max(0.0, axis_value - alpha * gap)
```

Worked example (king at -7 pp gsm8k, ref at 0.50, alpha=1.5):

```
raw math_bench  = 0.43
ref math_bench  = 0.50
gap             = 0.07
dock            = 1.5 * 0.07 = 0.105
adjusted        = 0.43 - 0.105 = 0.325
```

Since `worst()` is the primary ranking key, a king that's below base on any
bench axis sees that axis dominate min and lose ranking. A challenger that
ties or beats base keeps full credit.

### 2. KL probe skill diversification (private_pool.py + state JSON)

| Aspect | Before | After |
|--------|--------|-------|
| `DEFAULT_PRIVATE_FRACTION` | 0.10 | 0.20 |
| Pool size | 198 prompts | 278 prompts |
| Math / GSM8K-style | ~83 trivia ("how many" / arithmetic facts) | ~83 + 20 multi-step word problems |
| Code / HumanEval-style | 3 | 3 + 20 algorithm-completion prompts |
| Reasoning / BBH-style | scattered | 20 dedicated multi-step deductions |
| IFEval-style structure | ~82 | 82 + 20 with explicit constraints |

A model that has dropped GSM8K-style multi-step reasoning will produce
logits that diverge from the teacher's on the 20 added math word problems —
this directly raises KL on those prompts, which now make up ~20 % of every
KL probe. Same for code, reasoning, and structured instruction following.

Pool integrity is preserved by the existing private-pool commit/reveal:
- Pre-eval: validator publishes `sha256(sorted(per-prompt sha256))` to
  `state/private_pool_commit.json`.
- Post-eval: `state/private_pool_reveal.json` lists per-prompt hashes
  used in the round.
- Miners can verify the validator did not retro-fit the subset to the
  observed scores by replaying the commit.

## Expected effect

### Direct effects (next 1-2 rounds)

- Every current candidate that's below Qwen-4B-base on a bench axis sees
  that axis docked. The `composite.worst` of below-base kings drops by
  `α × max_gap` (typically 0.10-0.15 nats). The king is more likely to be
  dethroned by any challenger that's at-parity or above.
- The reference (UID -1) is unaffected (it's the anchor) — it's not eligible
  to be king anyway, but its role as a paired-comparison ground truth is
  preserved.

### 2nd-order effects (within 1 week)

- Miners optimizing the validator now have a CONCRETE incentive to retain
  Qwen-4B-base's reasoning/code/instruction skills, not just match the
  teacher on ClimbMix continuations. The straightforward gradient: distill
  from teacher AND keep base-model parity on math/code/reasoning. This
  narrows the design space toward a Pareto frontier where progress means
  beating base, not just beating prior kings.
- The held-out canary streak should converge: as new submissions retain
  base-level skills (forced by the new penalty), `king_canary_streak`
  resets and the dethrone gate stops firing on canary. The dashboard
  Goodhart-divergence flag (Canary strip) should clear.

### Risks

- **Risk: the reference itself fails an axis (eval-setup bug).** In that
  case all students "regress" relative to a pathological 0.0 baseline, and
  the penalty would dock everyone severely. Mitigation: the existing
  `resolve_reference_broken_axes` already drops axes where the reference
  scored 0/n; the broken axis gets removed from `worst()` BEFORE the
  penalty is applied, so the dock can't fire on a broken setup.
- **Risk: small reference samples (typical n=4-12 per axis) → noisy ref
  scores.** A round where the ref happens to score 0.75 on an axis it
  usually scores 0.50 on would over-penalize legit students. Mitigation:
  the `worst()` aggregation already smooths across multiple axes; the
  `n_docked` telemetry in `composite.baseline_penalty.applied` makes
  noisy single-axis docks visible. We can dial `α` down if the dashboard
  shows excess churn.
- **Risk: zero models pass after the penalty.** Possible if the broad
  pre-existing regression is universal. Mitigation: `worst()` still has the
  pre-existing `COMPOSITE_DETHRONE_FLOOR = 0.20` veto at promotion time,
  AND `KING_CANARY_GATE` waives the floor when the king is canary-regressing
  for 2+ rounds. So the gate degrades gracefully — if literally every
  candidate is below-base, the king holds (no churn) until someone
  improves.
- **Risk: KL probe wider variance from skill prompts.** Math word problems
  may have higher per-token entropy than ClimbMix continuations, which
  could raise KL absolute values across the board. Mitigation: KL is
  always normalized vs the round's king (via `_axis_kl`), so absolute
  shifts cancel; only relative differences matter. Empirically the new
  prompts are ~110 chars on average (matching the existing pool's mean),
  so token counts and KL magnitudes are comparable.
