### 1. Group-axis broken-axes handling fix (correctness)

Problem (v30.2): when a sub-axis was in `broken_axes` (the reference
4B scored 0 — eval-setup signal), the group axis still averaged it
in. So a student that scored badly on a broken sub-axis got
penalized through the group; a student that scored well on a broken
sub-axis got an inflated group score. Both directions break the v28
"broken axes don't penalize" invariant.

Fix: `_axis_skill_group_mean` now accepts a `broken_axes` set and
excludes any sub-axis in it from the mean. `compute_axes` accepts
the set and threads it through. `compute_composite` and
`annotate_h2h_with_composite` pass `broken` through to `compute_axes`
so all callers respect the invariant.

Edge cases preserved:
- Reference row uses `broken_axes=None` (it's the row that DEFINES
  what's broken — the reference is the anchor).
- All other student rows, king, and challenger axis computations
  pass the resolved `broken` set so their group axes drop broken
  sub-axes consistently.

Tests: 4 new tests in `TestGroupAxisBrokenHandling` (115 total pass).

### 2. Tail-decoupled KL shadow axis (synthesis §3.9 / §4.2 #3)

New shadow axis `tail_decoupled_kl` that splits the teacher's top-K
cache into HEAD (top-32 of top-128) and TAIL (positions 33..128) and
computes the IS-corrected KL contribution from each:

```
KL_head = Σ_{k ∈ top_32} p_t_full(k) · (log p_t - log p_s)
KL_tail = Σ_{k ∈ pos_33..128} p_t_full(k) · (log p_t - log p_s)
```

Per the Tail-Aware Distillation paper, "match teacher head but
flatten tail" is a documented SFT-only over-confidence pathology —
the student concentrates probability on the most-likely tokens and
misses the broad coverage the teacher places on rarer tokens. The
existing `kl` and `kl_is` axes are dominated by head contribution; a
poorly-trained student can score well on those while having terrible
tail coverage. The tail-decoupled signal exposes that gap.

Per-prompt fields: `kl_head`, `kl_tail`, `tail_mass` (teacher's
probability mass in the tail). Aggregated to model-level
`kl_head_mean`, `kl_tail_mean`, `tail_mass_mean`.

Composite axis: `_axis_tail_decoupled_kl` normalises `kl_tail_mean`
against the round-wide minimum (king-min). Default weight 0
(SHADOW); promote via `TAIL_DECOUPLED_KL_WEIGHT > 0` once correlation
telemetry validates the signal.

  per-prompt loop (variable lengths, attention masks, memory budget,
  batching all the EOPD/IS-KL/forking/trace metrics together).

  requires provisioning a multi-GPU pod with K2.6 weights pre-staged.
- **Pairwise judge tournament / OOD probe / adaptive eval / consensus**.
  Skipped per the user's audit instructions.
